# 03 门控注意力增强

这一步开始把主线从“粗双 actor 切换”收敛到“局部交互感知的轻量增强”。

## 这一步回答什么

一句话：

**不是直接切整个策略，而是在强交互局部状态下，只给原有策略补一层轻量交互增强。**

## 为什么转到这里

前面已经看到：

1. 单一 actor 继续覆盖训练不划算
2. 不同阶段策略确实有能力偏向
3. 但粗双 actor 切换两边都没赢

所以现在的判断是：

**问题不只是“要不要多策略”，而是“能不能识别当前交互状态，并只在需要时增强交互建模”。**

## 当前最小主线

- 保留原有主干
- 加一个轻量邻居注意力头
- 再加一个本地门控信号
- 只在强交互时提高注意力分支权重

## 当前建议顺序

1. 先做 attention critic
2. actor 先不大动
3. 先验证 dense 是否更稳
