# 保留专门化故事版

## 1. 我们一开始想解决什么

目标一直没变：

**把已经学会普通导航的多机器人策略，扩展到更复杂的 dense 多车交互场景。**

最早的直觉很直接：

- 先用课程学习把普通导航能力训出来
- 再继续往更复杂的 dense 场景训练
- 希望同一个 actor 最后同时兼顾普通导航和密集协同

## 2. 前面哪部分是成功的

前两段课程学习是有效的：

- 单车补稳有效
- 普通多车主线回接有效

这说明一件事：

**从简单到复杂逐步训练，这个大方向本身没有错。**

在这条线上，我们后来得到了一批关键节点：

- `5A`：普通五车 actor
- `5D`：带几何邻域 critic 的五车 actor

其中：

- `5A` 更像普通导航主干
- `5D` 没有彻底超过 `5A`，但它带了一些更强的交互基础

## 3. 真正卡住的地方是什么

问题出现在更复杂的 dense 多车阶段。

我们先试过更原始的五车 dense 课程，例如：

- `stage2_dense_gentle`
- `stage2_dense_bridge`

结果都不理想。

核心现象是：

- 同步五车中心强交互太难
- 共享 policy、局部观测、无通信时，车很容易一起冲向冲突区
- 即使 reward 修过，actor 继续更新后还是容易退化

所以旧的 pure dense 五车主线，基本走不顺。

## 4. 为什么后来改成 pair / three

因为直接做 pure dense 五车太硬，所以我们没有继续在那条路上死磕，而是把 dense 交互拆开。

新的想法是：

- 不再从“同步五车强对称中心冲突”起步
- 先做五车环境中的非对称双车主冲突
- 再做五车环境中的弱三车交互

对应就是：

- `stage3_asym_pair_5`
- `stage3_asym_three_5`

这不是换题。

更准确地说：

**这是把原来学不动的 pure dense 五车问题，拆成更可学的渐进 dense 版本。**

## 5. 这条新线里我们看到了什么

当前 `PAIR` 只保留 `PAIR(from_5d)`。它先从 `5D` 继承交互基础，再进入非对称双车冲突课程。

所以现在角色基本变成：

- `5A`：普通 actor
- `5D`：桥接 actor
- `PAIR(from_5d)`：当前 dense 专门化基线；正式 dense 测试未超过 `5D`

## 6. 那 single actor overwrite 为什么不行

把 `PAIR(from_5d)` 再往 `THREE_5` 继续训练后，仍然出现逐轮退化。

反复出现的现象都很像：

- warm start 模型本身还能跑
- 但 actor 一继续更新，就容易退化
- 有时普通场景能力先掉
- 有时 dense 场景也没有继续变强

这说明一个很关键的点：

**继续覆盖式训练同一个 actor，并不是纯增益。**

也就是说，普通导航能力和 dense 协同能力之间，至少存在明显的训练冲突。

## 7. 所以我们现在的判断是什么

现在的判断已经比较清楚：

1. pure dense 五车旧主线不顺
2. `PAIR / THREE_5` 这种渐进 dense 课程更合理
3. `5D -> PAIR` 这条线是目前最健康的 dense 专门化链路
4. 但继续把同一个 actor 往更复杂 dense 场景硬训，仍然容易退化

所以现在最自然的思路不是：

- 继续硬改同一个 actor

而是：

- 保留已经学好的普通 actor
- 保留 bridge actor
- 单独准备 dense 专门化 actor
- 然后只在强交互局部状态下做增强，而不是全局重写整个策略

## 8. 现在这条线怎么讲最顺

现在最顺的故事不是“双 actor 切换很新”。

更顺的讲法是：

**我们先证明了直接继续训练单一 actor 不稳定；然后发现不同阶段得到的 actor 确实有不同能力偏向；因此最终不再追求把同一个 actor 硬训成全能，而是转向 preserve-and-specialize。**

具体说：

- `5A` 保留普通导航能力
- `5D` 作为普通到 dense 的桥接基线
- `PAIR(from_5d)` 作为当前 dense 专门化基线

## 9. 当前测试口径

为了避免后面越做越乱，现在测试口径固定成三层：

- `standard_5`
  - 普通场景主测试
- `stage3_asym_three_5`
  - dense 场景主测试
- `stage2_dense`
  - 只作为压力测试，不作为当前主 benchmark

这里特别说明：

- `stage3_asym_three_5` 不是唯一 dense 场景
- 但它是当前这条主线里最合理、最稳定、最能比较模型能力的 dense benchmark

## 10. 之前做过什么，接下来还要做什么

### 已经做过的

- 普通导航课程主线已经接回五车
- 旧 pure dense 五车主线已经验证过，整体不顺
- `5D -> PAIR(from_5d)` 已完成
- `PAIR -> THREE_5` 的继续训练也做过，结果说明继续硬训会退化
- 粗双 actor 切换做过，结果不够好

### 最近完成的

- 已完成 `PAIR(from_5d)` 正式 test
  - `standard_5`
  - `stage3_asym_three_5`

### 下一步准备做的

在 `PAIR(from_5d)` 基线锁定之后，主线进一步收成：

1. 冻结当前表现最稳的 `5D` Actor。
2. 使用本车激光扇区和最近观测历史训练时空 Attention。
3. Attention 只输出有上限的门控残差，不替换基础 Actor。
4. 在同一训练分布中混合 standard、pair 和 three，避免顺序覆盖遗忘。
5. 如果最小 Attention 结构有效，再考虑单一交互奖励或安全控制分支。

`5A + 5D` 已完成上述前置检查：hard switch 没有超过 `5D`，三个 oracle case 也全部选择 `5D`，因此这组组合暂不进入 learned gate 训练。

## 11. 导师反馈之后的当前版本

旧 Attention 已精简，双 Actor 门控也因专家互补不足而停止。当前版本重新设计为冻结单个 `5D`，只学习本地可观测的时空 Attention 残差。

## 12. 现在这一版最简短的一句话

**冻结 `5D` 保留基础导航，Attention 只在本车时空观测上学习有限残差，并用混合课程同时约束 standard 与 dense 能力。**
